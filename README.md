# Приложение — что строим

## Концепция

Инструмент для занятий репетитора с учеником.
Не платформа для самостоятельного обучения — а среда для работы вдвоём.

---

## Ключевые функции (MVP)

### 1. Банк задач
- Задачи по темам и типам заданий ОГЭ
- Фильтр: тема / сложность / тип (задание 1–25)
- Режим «случайная задача по теме»

### 2. Пробный вариант
- Генерация полного варианта (25 заданий)
- Таймер (235 минут или укороченный)
- Ввод ответов (Часть 1) + загрузка фото (Часть 2)

### 3. Прогресс ученика
- Карточка ученика
- История вариантов с результатами
- Карта пробелов по темам

### 4. Конспекты
- Теория по каждой теме (из content/)
- Быстрый доступ с урока

---

## Что НЕ в MVP

- Автоматическая проверка Части 2 — это делает репетитор
- Геймификация, бейджи — не нужно
- Родительский кабинет — потом
- Онлайн-доска для решения задач — потом

| Вопрос | Варианты | Статус |
|--------|---------|--------|
| Платформа | Web / мобильное | не решено |
| Фронтенд | React / Vue / Svelte | не решено |
| Бэкенд | нужен? / только фронт | не решено |
| БД | Firebase / Supabase / локально | не решено |
| Хостинг | Vercel / другое | не решено |

→ Решения фиксируются в `DECISIONS.md`

---

## Что решить перед стартом разработки

1. Веб или мобильное? (мобильное удобнее для ученика на уроке)
2. Нужен ли бэкенд или достаточно локального хранилища?
3. Как вводятся задачи в банк — вручную или импорт из ФИПИ?
4. Математические формулы — MathJax / KaTeX?

---

## Приоритет разработки

1. Статичный банк задач (без бэкенда) — можно сделать быстро
2. Генератор варианта из банка
3. Прогресс ученика (локальное хранение)
4. Потом — бэкенд, если нужен
