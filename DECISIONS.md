## Ограничения и контекст

- Разрабатывает один человек (соло)
- Приоритет: работающий MVP быстро, не идеальная архитектура
- Аудитория приложения: репетитор + ученики 9 класса
- Устройства учеников: скорее всего смартфоны

## Стек-кандидаты

### Вариант A — простой веб (рекомендуется для старта)
- **Фронтенд:** React + Vite
- **Стили:** Tailwind CSS
- **Хранение:** localStorage (без бэкенда)
- **Формулы:** KaTeX
- **Хостинг:** Vercel (бесплатно)
- **Плюс:** запустить за 1–2 дня

### Вариант B — с бэкендом (для прогресса учеников)
- **Фронтенд:** React + Vite
- **Бэкенд:** Supabase (PostgreSQL + auth из коробки)
- **Хостинг:** Vercel + Supabase
- **Плюс:** хранение истории, несколько учеников
- **Минус:** дольше настраивать

### Вариант C — мобильное
- **Фреймворк:** React Native / Expo
- **Плюс:** привычнее для учеников
- **Минус:** деплой в стор, сложнее разработка
